# Building a Frappe App with a Custom Vue Frontend

> How to create a Frappe app that has its **own custom UI** (like Helpdesk at `/helpdesk/`) instead of using Frappe Desk's auto-generated forms (at `/app/`).

---

## Why Not Just Use Frappe Desk?

**Frappe Desk** (`/app/`) auto-generates forms from DocType definitions. It's great for admin CRUD, but limited in UX. You can't build custom dashboards, ticket views, or anything with a non-standard layout.

**A custom Vue frontend** gives you full control — your own layouts, components, routing, state management — while still using Frappe's backend for data, auth, permissions, and real-time updates.

The pattern: **Vue SPA hosted inside Frappe's website layer**. Two UIs, one backend.

---

## Architecture Overview

```
Browser requests /myapp/tickets
        |
        v
Frappe's website_route_rules catches /myapp/*
        |
        v
Renders www/myapp/index.html (Jinja template)
Injects boot data: csrf_token, session_user, site_name
        |
        v
Vue app bundle loads from /assets/myapp/desk/
Vue Router matches /tickets, renders component
        |
        v
Component calls Frappe API via frappe-ui (createResource, etc.)
        |
        v
Frappe backend handles request, returns JSON
```

## Step 4: `desk/vite.config.js` (The Build Pipeline)

This is the **most important file**. The `frappe-ui/vite` plugin handles the entire Frappe integration.

```js
import vue from "@vitejs/plugin-vue";
import frappeui from "frappe-ui/vite";
import path from "path";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [
    frappeui({
      // DEV: Proxy API calls from localhost:5173 → localhost:8000
      frappeProxy: true,

      // BUILD: Inject server boot data (csrf_token, etc.) into HTML
      jinjaBootData: true,

      // BUILD: Where to put compiled assets
      buildConfig: {
        outDir: `../myapp/public/desk`,
        emptyOutDir: true,
        indexHtmlPath: "../myapp/www/myapp/index.html",
      },
    }),

    vue(),
  ],

  resolve: {
    alias: {
      "@": path.resolve(__dirname, "src"),
    },
  },
});
```

### What each option does

**`frappeProxy: true`**
During development (`yarn dev`), Vite runs on `localhost:5173`. Your Vue app makes API calls to `/api/method/...`. The proxy intercepts these and forwards them to the Frappe backend (`localhost:8000`), so you don't get CORS errors. It reads `common_site_config.json` to find the port.

Proxied routes: `/app`, `/login`, `/api`, `/assets`, `/files`, `/private`

**`jinjaBootData: true`**
During production build, the plugin injects a Jinja block into your `index.html`:
```html
<script>
  {% for key in boot %}
  window["{{ key }}"] = {{ boot[key] | tojson }};
  {% endfor %}
</script>
```
This is how the server passes `csrf_token`, `session_user`, `site_name` to the frontend without an extra API call.

**`buildConfig`**
- `outDir`: JS/CSS bundles go to `myapp/public/desk/` (Frappe serves static files from `/assets/myapp/desk/...`)
- `indexHtmlPath`: The compiled HTML (with hashed script tags) is copied to `myapp/www/myapp/index.html` — this becomes the Jinja template Frappe renders

## Step 7: `desk/src/main.js` (App Bootstrap)

```js
import { createApp } from "vue";
import { createPinia } from "pinia";
import { FrappeUI, setConfig, frappeRequest } from "frappe-ui";

import App from "./App.vue";
import { router } from "./router";

import "./index.css";

// Tell frappe-ui to use frappeRequest for all API calls
// This handles CSRF tokens, cookies, error handling automatically
setConfig("resourceFetcher", frappeRequest);

const app = createApp(App);
const pinia = createPinia();

app.use(FrappeUI);   // Registers all frappe-ui components globally
app.use(pinia);       // State management
app.use(router);      // Client-side routing

// In dev mode, boot data isn't in the HTML (no Jinja rendering)
// So we fetch it via API before mounting
if (import.meta.env.DEV) {
  frappeRequest({
    url: "/api/method/myapp.www.myapp.index.get_context_for_dev",
  }).then((values) => {
    for (let key in values) {
      window[key] = values[key];
    }
    app.mount("#app");
  });
} else {
  // Production: boot data already injected by Jinja template
  app.mount("#app");
}
```

### What's happening here

1. **`setConfig("resourceFetcher", frappeRequest)`** — This is critical. All `createResource()`, `createDocumentResource()`, `createListResource()` calls will use `frappeRequest`, which automatically:
   - Adds `X-Frappe-CSRF-Token` header from `window.csrf_token`
   - Sends cookies for session auth
   - Handles error responses

2. **Dev vs Production boot** — In production, the server renders the HTML with boot data already in `<script>` tags (`window.csrf_token = "..."`, etc.). In dev, the HTML is served by Vite (no Jinja), so we fetch the same data via an API call.

## Step 10: `desk/src/router/index.ts` (Vue Router)

```ts
import { createRouter, createWebHistory } from "vue-router";

const routes = [
  {
    path: "/",
    name: "Home",
    component: () => import("@/pages/Home.vue"),
  },
  {
    path: "/tickets",
    name: "Tickets",
    component: () => import("@/pages/Tickets.vue"),
  },
  {
    path: "/tickets/:ticketId",
    name: "TicketDetail",
    component: () => import("@/pages/TicketDetail.vue"),
    props: true,
  },
  {
    path: "/customers",
    name: "Customers",
    component: () => import("@/pages/Customers.vue"),
  },
  {
    path: "/settings",
    name: "Settings",
    component: () => import("@/pages/Settings.vue"),
  },
];

export const router = createRouter({
  // IMPORTANT: Base path must match your app's URL prefix
  history: createWebHistory("/myapp/"),
  routes,
});

// Optional: Auth guard
router.beforeEach((to, from, next) => {
  // window.session_user is set by boot data
  if (window.session_user === "Guest") {
    window.location.href = `/login?redirect-to=/myapp${to.fullPath}`;
  } else {
    next();
  }
});
```

**The base path `/myapp/` is key.** When you define a route as `/tickets`, the actual browser URL will be `/myapp/tickets`. Vue Router handles this transparently — `<router-link to="/tickets">` generates `/myapp/tickets`.

### `myapp/www/myapp/index.py`

This file provides boot data to the Jinja template. It runs server-side when someone visits `/myapp`.

```python
import frappe
from frappe.utils import cint

no_cache = 1  # Always serve fresh (don't cache the HTML)


def get_context(context):
    """Called by Frappe when rendering the page. Injects boot data."""
    frappe.db.commit()
    context.boot = get_boot()
    return context


@frappe.whitelist(methods=["POST"], allow_guest=True)
def get_context_for_dev():
    """Called by the Vue dev server to get boot data (dev mode only)."""
    if not frappe.conf.developer_mode:
        frappe.throw("This method is only for developer mode")
    return get_boot()


def get_boot():
    """Server-side data passed to the frontend."""
    return frappe._dict({
        "default_route": "/myapp",
        "site_name": frappe.local.site,
        "csrf_token": frappe.sessions.get_csrf_token(),
        "session_user": frappe.session.user,
        "favicon": "/assets/myapp/desk/favicon.svg",
        "setup_complete": cint(frappe.get_system_settings("setup_complete")),
    })
```

## Step 13: Backend — `hooks.py` (App Registration)

Add these to your `myapp/hooks.py`:

```python
app_name = "myapp"
app_title = "My App"
app_publisher = "Your Name"

# 1. Register in Frappe's app switcher sidebar
add_to_apps_screen = [
    {
        "name": "myapp",
        "logo": "/assets/myapp/desk/favicon.svg",
        "title": "My App",
        "route": "/myapp",
        "has_permission": "myapp.api.permission.has_app_permission",
    }
]

# 2. Route ALL /myapp/* URLs to the same page (SPA catch-all)
website_route_rules = [
    {
        "from_route": "/myapp/<path:app_path>",
        "to_route": "myapp",
    },
]
```

### What `website_route_rules` does

Without this rule:
- `/myapp` works (matches `www/myapp/index.html`)
- `/myapp/tickets` returns 404 (no `www/myapp/tickets/` folder exists)

With this rule:
- `/myapp/anything/at/all` → renders `www/myapp/index.html`
- Vue Router then reads the URL and shows the right component

This is the standard pattern for hosting SPAs — the server always returns the same HTML, and the client handles routing.

### `add_to_apps_screen`

This adds your app's icon to the Frappe sidebar (the grid of apps when you click the app switcher). When clicked, it navigates to `/myapp`.

```python
"has_permission": "myapp.api.permission.has_app_permission"
```

Create this function to control who sees the app:

```python
# myapp/api/permission.py
import frappe

def has_app_permission():
    return "System Manager" in frappe.get_roles()
```


## How frappe-ui Talks to the Backend

frappe-ui provides several patterns for data fetching. All of them use `frappeRequest` under the hood, which handles auth (CSRF token + session cookie) automatically.

### `createResource` — Call any API endpoint

```js
import { createResource } from "frappe-ui";

const resource = createResource({
  url: "myapp.api.dashboard.get_stats",  // Python dotpath
  method: "GET",                          // or "POST"
  params: { status: "Open" },            // Query params
  auto: true,                            // Fetch on mount
  onSuccess(data) {
    console.log("Got:", data);
  },
  onError(err) {
    console.error("Failed:", err);
  },
});

// Reactive properties:
// resource.data     — response data
// resource.loading  — boolean
// resource.error    — error object
// resource.fetch()  — manual refetch
// resource.reload() — refetch with same params
```

### `createDocumentResource` — CRUD for a single document

```js
import { createDocumentResource } from "frappe-ui";

const customer = createDocumentResource({
  doctype: "HD Customer",
  name: "CUST-001",
  auto: true,    // Fetch the document on mount
});

// Read:
customer.doc.customer_name;  // "Tesla Inc."

// Update:
await customer.setValue.submit({
  email: "new@tesla.com",
  phone: "+1 555 0123",
});

// Delete:
await customer.delete.submit();
```

## Adding Real-Time Updates (WebSocket)

Frappe has built-in Socket.io support. To use it:

### `desk/src/socket.js`

```js
import { io } from "socket.io-client";

export function initSocket() {
  let host = window.location.hostname;
  let port = window.location.port ? `:${window.location.port}` : "";
  let protocol = port ? "http" : "https";
  let url = `${protocol}://${host}${port}/${window.site_name}`;

  return io(url, {
    withCredentials: true,
    reconnectionAttempts: 5,
  });
}
```

### In `main.js`

```js
import { initSocket } from "./socket";

const socket = initSocket();
app.config.globalProperties.$socket = socket;
```

### In a component

```vue
<script setup>
import { getCurrentInstance, onMounted, onUnmounted } from "vue";

const { proxy } = getCurrentInstance();
const socket = proxy.$socket;

onMounted(() => {
  socket.on("helpdesk:ticket-updated", (data) => {
    console.log("Ticket updated:", data);
    // Refetch data, show notification, etc.
  });
});

onUnmounted(() => {
  socket.off("helpdesk:ticket-updated");
});
</script>
```

### Emit from Python

```python
import frappe

frappe.publish_realtime(
    "helpdesk:ticket-updated",
    {"ticket": "TKT-001", "status": "Closed"},
    user=frappe.session.user,
)
```

## Adding a Pinia Store (State Management)

For shared state across components:

`desk/src/stores/auth.ts`:

```ts
import { defineStore } from "pinia";
import { createResource } from "frappe-ui";
import { computed, ref } from "vue";

export const useAuthStore = defineStore("auth", () => {
  const userResource = createResource({
    url: "myapp.api.auth.get_user",
  });

  const isLoggedIn = computed(() => {
    return window.session_user && window.session_user !== "Guest";
  });

  const user = computed(() => userResource.data);

  async function init() {
    if (!userResource.fetched) {
      await userResource.fetch();
    }
  }

  return { isLoggedIn, user, init };
});
```

Use in any component:
```vue
<script setup>
import { useAuthStore } from "@/stores/auth";

const auth = useAuthStore();
await auth.init();
</script>

<template>
  <div v-if="auth.isLoggedIn">
    Welcome, {{ auth.user?.full_name }}
  </div>
</template>
```

## Quick Reference: The Request Flow

```
Vue Component
    |
    | createResource({ url: "myapp.api.foo.bar", params: {...} })
    v
frappe-ui (frappeRequest)
    |
    | POST /api/method/myapp.api.foo.bar
    | Headers: X-Frappe-CSRF-Token: <token>
    | Cookies: user_id=<session>, sid=<session>
    v
Frappe Backend
    |
    | Routes to myapp/api/foo.py → bar()
    | Checks @frappe.whitelist() permissions
    | Executes Python, returns JSON
    v
frappe-ui
    |
    | Updates resource.data (reactive)
    | Triggers onSuccess/onError callbacks
    v
Vue Component re-renders
```

## Common Gotchas

### 1. 404 on page refresh
If `/myapp/tickets` gives a 404 on page refresh, your `website_route_rules` in `hooks.py` is missing or wrong. Make sure it catches `<path:app_path>`.

### 2. CSRF token errors
If API calls fail with "CSRF token missing", check:
- `jinjaBootData: true` in vite.config.js
- `get_boot()` returns `csrf_token`
- `setConfig("resourceFetcher", frappeRequest)` in main.js

### 3. Dev server can't reach Frappe
Make sure `bench start` is running. The Vite proxy reads `common_site_config.json` for the port. Default is 8000.

### 4. Build output not showing
After `yarn build`, run `bench build` to copy assets to the sites directory. Then clear cache: `bench clear-cache`.

### 5. `index.html` in `www/` is empty
You need to run `yarn build` first. The build process generates this file. Never edit it manually.

### 6. Boot data not available in dev
Make sure `get_context_for_dev()` exists in your `index.py` and `frappe.conf.developer_mode` is enabled in your `site_config.json`.

| App sidebar | `hooks.py` → `add_to_apps_screen` | Icon in Frappe's app switcher |
